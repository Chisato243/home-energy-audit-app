Home Energy Audit App

Summer Research Project Documentation

Abstract

This project explores the design and development of a simple mobile and web application that helps users conduct basic home energy audits. The tool guides people through a clear process to record house features, identify possible energy waste, and suggest improvements. The final product is designed to be accessible, structured, and easy to use for both personal learning and real evaluation tasks.

Motivation

Many homes consume more energy than necessary, but not everyone has access to professional energy audit services. Existing tools are often expensive, complex, or hard to use. We want to build a lightweight app that allows trained users to perform simple energy evaluations, helping them make informed decisions and reduce waste at home.

Background

Home energy audits usually involve checking areas like insulation, air leaks, lighting, and appliance efficiency. Utility data is also analyzed to see how energy is being used. Auditors often use specialized tools, but there is a need for more accessible solutions. This project looks into how such a process can be supported digitally, especially for students or individuals who want to learn and take action themselves.

Objectives

  • Build a step-by-step app for home energy evaluation using Flutter
  • Include sections for home info, audit checklist, utility data, sketches, and notes
  • Generate a summary report based on user input
  • Support use on both mobile and desktop browsers

System Design Overview

The app is divided into several screens: a welcome page, home information input, category-based audit checklists, an optional sketch tool, and a summary report screen. All data is stored locally at first. The interface is clean and responsive, with dark mode by default. The system is developed in Flutter and deployed using GitHub Pages for web compatibility.

Timeline

  • May: Research and design planning, UI sketches, environment setup
  • June: Core module development: home info, checklist, and navigation
  • July: Sketch tool, report output, visual polishing
  • August: Testing, documentation, and final demo preparation
